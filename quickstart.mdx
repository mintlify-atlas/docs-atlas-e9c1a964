---
title: "Get started with CS2 SDK: build and run your first dump"
description: "Build cs2-sdk.exe from source and run your first Counter-Strike 2 dump in two commands. Produces offsets, schemas, and signatures in under 15 seconds."
---

CS2 SDK ships as a single Rust binary you build from source. The steps below take you from a fresh clone to a completed dump folder. The entire process — build aside — takes around 12 seconds on a typical machine.

<Steps>
  <Step title="Check prerequisites">
    Before you start, confirm you have the following:

    - **Rust 1.85 or later** — install via [rustup.rs](https://rustup.rs)
    - **Windows 10 or 11** — the default memory backend (`memflow-native`) is Windows-only
    - **Administrator privileges** — required to attach to a running process
    - **CS2 open** — launch Counter-Strike 2 and leave it running at the menu or in-game
  </Step>
  <Step title="Clone and build">
    Clone the repository and build a release binary. The release profile enables LTO, stripping, and single-codegen-unit optimisation, producing a small, fast standalone executable.

    ```powershell
    git clone https://github.com/scros22/cs2-universal-offsets
    cd cs2-universal-offsets
    cargo build --release
    ```

    The binary is written to `target\release\cs2-sdk.exe`.
  </Step>
  <Step title="Run as Administrator">
    Open an elevated terminal (right-click → **Run as administrator**) and execute:

    ```powershell
    .\target\release\cs2-sdk.exe
    ```

    The tool will:

    1. Attach to `cs2.exe` via `memflow-native`
    2. Enumerate the CS2 modules it cares about
    3. Run the offsets, schema, interfaces, and buttons pass
    4. Run the signature pass
    5. Write everything into a dated session folder
    6. Play short audio cues for start, progress, success, and failure (skip with `--no-sound`)
  </Step>
  <Step title="Find your output">
    After a successful run, the output directory (default: `output\`) contains three subdirectories and a manifest:

    ```text
    output\
      manifest.json
      sdk\
      signatures\
      logs\
    ```

    For most C++ cheat projects, **`sdk\cs2sdk.hpp`** is the single file you need — it is a single-include header with typed `SCHEMA_FIELD` accessors, resolved interfaces, netvars, and signature constants. See [Output layout](/output-layout) for a full file-by-file reference.
  </Step>
</Steps>

<Warning>
  You must run `cs2-sdk.exe` from an elevated (Administrator) terminal. If CS2 is not running when you execute the tool, it will exit cleanly with a "process not found" message — no output folder is created.
</Warning>

## Common CLI examples

<CodeGroup>

```powershell Basic run (defaults)
.\target\release\cs2-sdk.exe
```

```powershell Custom output directory with verbose logging
.\target\release\cs2-sdk.exe --output D:\cs2-sdk -vv
```

```powershell Signatures pass only
.\target\release\cs2-sdk.exe --skip-offsets
```

```powershell Offsets pass only
.\target\release\cs2-sdk.exe --skip-signatures
```

```powershell Warm cache from a previous session
.\target\release\cs2-sdk.exe --cache .\dumps\25-04-26-CS2-SDK\signatures\signatures.json
```

</CodeGroup>

<Info>
  By default, output is written to an `output\` folder in the current directory. Use `--output D:\my-dump` to specify a different location. Re-running the tool overwrites the previous output in that directory.
</Info>

<Tip>
  Add `--no-sound` to silence the audio cues if you're running in a headless or quiet environment: `.\cs2-sdk.exe --no-sound`.
</Tip>
